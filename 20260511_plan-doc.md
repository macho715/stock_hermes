## B. Context & Requirements

### B1. 문제 정의
현재 대시보드는 연구 스크리닝 도구(screening only)로 설계되어 있다.
실제 투자 결정을 지원하려면 3가지 격차를 해소해야 한다:
1. **실시간 시각화 부재**: 정적 Recharts로는 틱 단위 가격 흐름 파악 불가
2. **단일 ML 모델**: LightGBM의 선형 특성 포착 한계 → Transformer 보완 필요
3. **advisory_score와 포트폴리오 최적화 미연결**: LLM 의견이 BL 가중치에 반영되지 않음

### B2. 페르소나
| 페르소나 | 니즈 |
|----------|------|
| 개인 퀀트 투자자 (사용자 1명) | 실시간 시세, 알고리즘 신호, 포트폴리오 최적화 가중치를 한 화면에서 확인 |

### B3. User Stories
| ID | As a... | I want... | So that... | Priority |
|----|---------|-----------|------------|----------|
| US-1 | 투자자 | 실시간 캔들차트를 보고 싶다 | 매수/매도 타이밍을 시각적으로 판단할 수 있다 | P0 |
| US-2 | 투자자 | Transformer + LightGBM 앙상블 신호를 보고 싶다 | 단일 모델 대비 더 높은 신뢰도로 방향성을 예측할 수 있다 | P1 |
| US-3 | 투자자 | LLM 의견이 반영된 BL 포트폴리오 가중치를 보고 싶다 | 정성적 뷰와 정량적 최적화를 통합한 자산배분을 받을 수 있다 | P1 |
| US-4 | 투자자 | 추천 결과가 스크리닝 전용임을 명확히 알고 싶다 | 시스템이 자동으로 주문을 실행하지 않음을 확인할 수 있다 | P0 (safety) |

### H2. 신규 테스트 파일 스펙

- `test_forward_shape()`: `TransformerTimeSeries(input_dim=20).forward(B=4, T=60, F=20)` → shape `(4, 3)`

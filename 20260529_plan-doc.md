## A. Executive Summary

### A1. Goal

세 가지 핵심 이니셔티브로 `stock_rtx4060`의 규정 준수·관측 가능성·데이터 다양성을 단계적으로 강화한다.

1. **GAP Surgical Fixes (P0)** — PHASE1_GAP_ANALYSIS에서 확인된 5개 spec 위반을 `paper_trading.py`에서 수정하고 대응 테스트 5개를 추가.
2. **MLflow 3.x Dataset Tracking (P3)** — `mlflow.log_input()` + 멀티-체크포인트 로깅으로 모든 학습 실험에 데이터셋 provenance와 체크포인트 아티팩트를 기록.
3. **OpenBB MCP Macro Data (P1/P6)** — OpenBB MCP endpoint를 통해 FRED·IMF·BLS·FOMC 매크로 데이터를 인제스트하고 LLM 어드바이저의 도구로 노출.

추가로 **Chaos Engineering Surprise Pick (BONUS)** — paper trading 루프에 합성 장애를 주입해 리스크 게이트가 실제로 동작하는지 검증하는 adversarial 테스트 suite를 작성한다.

```python
# src/stock_rtx4060/data_lake/ingest/openbb_ingestor.py  (신규 파일 전체)
"""OpenBB MCP → PIT lake ingestor + MacroRegimeAgent panel_fetcher."""
from __future__ import annotations

import logging
import os
from typing import Any

import pandas as pd

logger = logging.getLogger(__name__)


class OpenBBMCPIngestor:
    """Thin wrapper over OpenBB MCP HTTP endpoint.

    Graceful skip when OPENBB_MCP_URL not set or USE_OPENBB_MCP=0.
    PIT-correct: as_of param raises RuntimeError on unavailability.
    """

    DEFAULT_MCP_URL: str = "http://localhost:6340"

    def __init__(self) -> None:
        self._base = os.getenv("OPENBB_MCP_URL", self.DEFAULT_MCP_URL)

    def is_available(self) -> bool:
        """Return False if USE_OPENBB_MCP=0 or server unreachable."""
        if os.getenv("USE_OPENBB_MCP", "1") == "0":
            return False
        try:
            import requests
            r = requests.get(f"{self._base}/health", timeout=3)
            return r.status_code == 200
        except Exception:
            return False

    def fetch_fred(
        self,
        series_id: str,
        as_of: str | None = None,
        limit: int = 252,
    ) -> pd.DataFrame:
        """Return FRED series as DataFrame with DatetimeIndex.

        Raises RuntimeError if as_of is set but server unavailable (PIT guard).
        Returns empty DataFrame silently when USE_OPENBB_MCP=0 and as_of is None.
        """
        if os.getenv("USE_OPENBB_MCP", "1") == "0":
            if as_of is not None:
                raise RuntimeError(
                    f"as_of={as_of!r} set but USE_OPENBB_MCP=0 — "
                    "cannot guarantee PIT correctness without OpenBB MCP"
                )
            return pd.DataFrame()

        if not self.is_available():
            if as_of is not None:
                raise RuntimeError(
                    f"as_of={as_of!r} set but OpenBB MCP unavailable at {self._base}"
                )
            logger.warning("OpenBB MCP unavailable — returning empty DataFrame")
            return pd.DataFrame()

        try:
            import requests
            params: dict[str, Any] = {"symbol": series_id, "limit": limit}
            if as_of:
                params["end_date"] = as_of
            r = requests.get(
                f"{self._base}/economy/fred",
                params=params,
                timeout=5,
            )
            r.raise_for_status()
            rows = r.json().get("results", [])
            if not rows:
                return pd.DataFrame()
            df = pd.DataFrame(rows)
            df["date"] = pd.to_datetime(df["date"])
            df = df.set_index("date").sort_index()
            return df
        except Exception as exc:
            logger.warning("OpenBB FRED fetch failed: %s", exc)
            return pd.DataFrame()

    def fetch_fomc_calendar(self, year: int) -> list[dict]:
        """Return FOMC meeting dates.

        Each dict: {date: str, type: str, decision: str | None}
        Returns [] if unavailable.
        """
        if not self.is_available():
            return []
        try:
            import requests
            r = requests.get(
                f"{self._base}/economy/calendar/fomc",
                params={"year": year},
                timeout=5,
            )
            r.raise_for_status()
            return r.json().get("results", [])
        except Exception as exc:
            logger.warning("OpenBB FOMC fetch failed: %s", exc)
            return []

    def as_panel_fetcher(self):
        """Return callable for MacroRegimeAgent.panel_fetcher injection.

        Usage:
            MacroRegimeAgent(panel_fetcher=OpenBBMCPIngestor().as_panel_fetcher())
        """
        def _fetch() -> dict[str, Any]:
            df_t10y2y = self.fetch_fred("T10Y2Y", limit=1)
            df_vix = self.fetch_fred("VIXCLS", limit=1)
            df_dxy = self.fetch_fred("DTWEXBGS", limit=1)
            fomc = self.fetch_fomc_calendar(pd.Timestamp.now().year)
            return {
                "t10y2y": float(df_t10y2y.iloc[-1, 0]) if not df_t10y2y.empty else None,
                "vix": float(df_vix.iloc[-1, 0]) if not df_vix.empty else None,
                "dxy": float(df_dxy.iloc[-1, 0]) if not df_dxy.empty else None,
                "fomc_next": fomc[0].get("date") if fomc else None,
                "kospi_trend_pct": None,
                "spy_trend_pct": None,
                "krwusd": None,
            }
        return _fetch
```

### Why This Works (Cross-Domain Rationale — Chaos Engineering)

Netflix의 Chaos Monkey는 "고장이 나도 시스템이 살아남는가"를 프로덕션에서 증명하기 위해 탄생했다. 금융 시스템에 같은 철학을 적용하면: 실제 리스크 게이트가 합성 장애(가격 스파이크 +40%, 시그널 프리즈, 체결 오류)에 대해 *설계된 대로 막는지* 코드 기반이 아닌 실행 기반으로 검증할 수 있다. 기존 backtesting은 과거 데이터를 재생하지만, Chaos Engineering은 역사에 없었던 장애 모드를 합성해 취약점을 드러낸다.

## J. Error Handling & Recovery

### J1. Error Taxonomy

| 레이어 | 에러 | 처리 |
|--------|------|------|
| GAP fix | score threshold 미충족 | `PaperDecision(gate_result="score_below_threshold")` 반환; 예외 없음 |
| GAP fix | force_rerun + no reason | `ValueError("rerun_reason required")` — 즉시 실패 |
| MLflow | `mlflow.log_input` 실패 | `logger.warning`; MLflow 에러가 학습 루프를 중단시키지 않도록 `try/except` |
| OpenBB | MCP server ConnectionError | `is_available()=False` 반환; 빈 DataFrame; 로그 경고 |
| OpenBB | PIT guard 위반 | `RuntimeError("as_of set but OpenBB MCP unavailable")` — 즉시 실패 |
| Chaos | 실험 중 예상치 못한 예외 | `ExperimentResult(exception=exc, passed=False)`; 테스트 FAIL |
| CI | 커버리지 < 75% | exit code 1; merge 블록 |

### J2. Retry Policy

- MLflow log_input: 재시도 없음 (로컬 SQLite, 실패 시 warning만)
- OpenBB HTTP fetch: 1회 재시도 (ConnectionError), 타임아웃 5초
- CI: 재시도 없음 (fail fast)

### J3. Idempotency

- GAP-02 timestamp: `PaperDecision` 생성 시 1회 세팅; 재직렬화 시 동일 값 유지
- MLflow log_input: 동일 run_id에 중복 호출 시 MLflow가 마지막 값으로 덮어씀 (safe)
- OpenBBMCPIngestor: stateless; 동일 인수로 호출 시 동일 결과 (PIT-correct)

### J4. Timeout

- OpenBB MCP fetch: `requests.get(timeout=5)`
- Chaos 실험: 단일 tick 기준 5초 이상 소요 시 `pytest-timeout` 10초 제한
- MLflow artifact upload: 로컬 파일시스템; 타임아웃 없음

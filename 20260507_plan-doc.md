## A. Executive Summary

### A1. Goal

Elevate `stock_rtx4060_unified` from a manually-verified research prototype to a measurably reliable, CI-gated codebase with:

1. **Automated quality gate** — every push and PR fails if coverage drops below threshold or any test regresses.
2. **Complete test coverage** for 5 currently-untested modules (alert engine, broker bridge, portfolio analytics, position tracker, trade journal).
3. **Local SQLite data cache** that eliminates repeat yfinance/pykrx network calls and makes the development loop fast and offline-capable.

### A2. KPIs

| KPI | Baseline (2026-05-07) | Target |
|-----|----------------------|--------|
| Test suite size | 133 tests | ≥ 200 tests |
| Coverage (line) | Unknown (no CI gate) | ≥ 80% gated in CI |
| CI run time (full suite) | No CI pipeline | < 3 min on GH Actions |
| Cold-start recommendation run | ~30–90 s (network) | < 10 s (cache hit) |
| `yfinance` calls on re-run | N per universe | 0 (cache hit, TTL 20 h) |
| Pre-existing test failures | 1 (`test_trade_journal`) | 0 |

### A3. Scope

**In scope:**
- GitHub Actions workflow (`.github/workflows/ci.yml`)
- `pyproject.toml` coverage configuration
- 5 new test files with ≥ 80% coverage of their target modules
- `src/stock_rtx4060/data_cache.py` — new SQLite cache module
- Integration hooks in `data_providers.py` (or equivalent data fetch call sites)
- Fix pre-existing `test_trade_journal` failure

**Out of scope:**
- Broker execution, live orders, account credentials
- Cloud deployment or hosted CI beyond GitHub Actions
- External Codecov / coverage badge service (optional, additive only)
- Changes to recommendation thresholds or model logic
- KRX pilot or Phase 2 paper trading

### A4. Core Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D-1 | `[tool.coverage.report] fail_under = 80` in `pyproject.toml` | Canonical location per cvxportfolio, LibCST, pystac patterns; no justfile dependency |
| D-2 | `pytest-cov` plugin + `coverage xml` + `codecov/codecov-action@v5` | Standard 2025 CI pattern; tokenless for public repo |
| D-3 | SQLite WAL mode, 20 h TTL, `USE_DATA_CACHE=0` kill switch | Matches diskcache/requests-cache patterns; WAL for concurrent reads |
| D-4 | `test_broker_bridge.py` tests simulation-boundary assertions only | Mirrors NautilusTrader Sandbox adapter + open-paper-trading-mcp QUOTE_ADAPTER_TYPE pattern |
| D-5 | Cache key = `(ticker, period, provider, fetch_date_utc)` | Deterministic; survives process restarts; avoids stale data from same-day multiple fetches |

## B. Context & Requirements

### B1. Problem Definition

### B3. User Stories

- As a developer, I want every `git push` to run all tests and fail if coverage drops, so I catch regressions immediately.
- As a developer, I want `python main.py recommend` to use cached OHLCV on re-run, so I don't wait for network calls during iteration.
- As a reviewer, I want test files for `alert_engine`, `broker_bridge`, `portfolio_analytics`, `position_tracker`, `trade_journal` so I can verify module boundaries.
- As a maintainer, I want `test_broker_bridge.py` to assert that broker calls raise `NotImplementedError` or return simulation-only records, so the paper-only boundary is machine-verifiable.

## C. UI/UX Plan

This initiative has no user-facing UI changes. The "UX" is the developer experience:

### C1. Developer UX Flows

**Flow 1 — Push to branch:**
```
git push → GitHub Actions triggered →
  setup-python → pip install → pytest --cov → coverage xml →
  codecov upload → [PASS: green check] or [FAIL: annotation on failing line]
```

**Flow 2 — Cache hit on re-run:**
```
python main.py recommend --universe AAPL,MSFT --period 3y
→ DataCache.get("AAPL", "3y", "yfinance") → HIT → returns DataFrame (< 50 ms)
→ DataCache.get("MSFT", "3y", "yfinance") → HIT → returns DataFrame (< 50 ms)
→ No network calls → recommendation output in < 5 s
```

**Flow 3 — Cache miss (first run or expired):**
```
→ DataCache.get("AAPL", "3y", "yfinance") → MISS →
→ yfinance.download() → success → DataCache.set(...) → continue
```

**Flow 4 — Kill switch:**
```
USE_DATA_CACHE=0 python main.py recommend ...
→ DataCache skipped entirely → direct yfinance/pykrx calls
```

### C2. CLI Output (cache status)

When `--verbose` is active, print cache status per ticker:
```
[CACHE HIT]  AAPL  3y  yfinance  fetched=2026-05-07T08:00:00Z  age=4h
[CACHE MISS] NVDA  3y  yfinance  fetching...
[CACHE SET]  NVDA  3y  yfinance  stored 756 rows
```

When not verbose: silent (no change to existing output format).

### C3. Screens / Views

Not applicable (CLI tool). The only "screen" change is the CI badge in README (optional, additive).

## E. Data Model & API Contract

### E1. Entity: `DataCache` (SQLite table)

```sql
CREATE TABLE IF NOT EXISTS ohlcv_cache (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    ticker        TEXT    NOT NULL,
    period        TEXT    NOT NULL,
    provider      TEXT    NOT NULL,
    fetch_date    TEXT    NOT NULL,  -- ISO-8601 UTC, e.g. "2026-05-07T08:00:00Z"
    row_count     INTEGER NOT NULL,
    payload_json  TEXT    NOT NULL,  -- JSON array of OHLCV dicts
    created_at    TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ','now')),
    UNIQUE(ticker, period, provider, fetch_date)
);

PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
```

Cache key: `(ticker, period, provider, fetch_date)` where `fetch_date` is truncated to the hour (e.g., `2026-05-07T08`).

### E2. Python API Contract

```python
class DataCache:
    DEFAULT_TTL_HOURS: int = 20
    DEFAULT_DB_PATH: Path = Path.home() / ".stock_rtx4060_cache.db"

    def __init__(self, db_path: Path = DEFAULT_DB_PATH, ttl_hours: int = DEFAULT_TTL_HOURS) -> None: ...

    def get(self, ticker: str, period: str, provider: str) -> Optional[pd.DataFrame]:
        """Return cached DataFrame or None if miss/expired/cache disabled."""

    def set(self, ticker: str, period: str, provider: str, df: pd.DataFrame) -> None:
        """Persist DataFrame; silently no-ops if cache disabled or df is empty."""

    def invalidate(self, ticker: str) -> int:
        """Delete all rows for ticker. Returns deleted row count."""

    def purge_expired(self) -> int:
        """Delete all expired rows. Returns deleted row count."""

    @staticmethod
    def is_enabled() -> bool:
        """Return False if USE_DATA_CACHE=0."""
```

### E3. CI Workflow Contract

```yaml
# .github/workflows/ci.yml  (key shape)
on:
  push:
    branches: ["main", "master"]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install -e ".[dev]" --quiet
      - run: pytest --cov=stock_rtx4060 --cov-report=xml --cov-report=term-missing -q
      - uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage.xml
          include-hidden-files: true
      - uses: codecov/codecov-action@v5
        with:
          file: coverage.xml
          fail_ci_if_error: false
```

```toml
# pyproject.toml additions
[tool.coverage.run]
source = ["src/stock_rtx4060"]

```

### I4. Runbook: Cache Corruption

Symptoms: `sqlite3.DatabaseError` or `json.JSONDecodeError` on cache get.

Steps:
1. `DataCache.get()` catches `Exception`, logs warning, returns `None` (fall-through to network).
2. If persistent: `python -c "from stock_rtx4060.data_cache import DataCache; DataCache().purge_expired()"`.
3. Nuclear option: `rm ~/.stock_rtx4060_cache.db` (safe, no data loss; only cached market data).

### I5. Runbook: CI Coverage Drop

Symptoms: CI fails with `FAIL Required test coverage of XX% not reached. Total coverage: YY%`.

Steps:
1. Check `coverage.xml` artifact for which files dropped.
2. Add tests for uncovered lines; or add `# pragma: no cover` for untestable branches (requires PR review comment).
3. Do not raise `fail_under` to compensate for dropped coverage without adding tests.

## J. Error Handling & Recovery

### J1. Error Taxonomy

| Layer | Error | Handling |
|-------|-------|---------|
| CI | Pytest import error | Fail fast; log full traceback |
| CI | Coverage < threshold | Exit code 1; annotation on PR |
| Cache | `sqlite3.OperationalError` | Log warning; return `None` (fall-through) |
| Cache | `json.JSONDecodeError` | Delete corrupt row; return `None` |
| Cache | `PermissionError` on DB path | Log error; disable cache for session |
| Test | `test_list_entries_filter` failure | Fix root cause (not suppress) |
| Data provider | Network timeout | Existing retry logic; cache miss treated as timeout |

### J2. Retry Policy

- Cache reads: no retry (either hit or miss; miss → network).
- Cache writes: 1 retry on `sqlite3.OperationalError` (WAL contention); if fails again, log warning and continue.
- CI: no retry on test failure (fail fast).

### J3. Idempotency

- `DataCache.set()` uses `INSERT OR REPLACE` — safe to call multiple times with same key.
- CI workflow runs are independently idempotent.
- Test files use `tmp_path` — no shared state between test runs.

### J4. Timeout

- CI job timeout: 10 minutes (GitHub Actions default; explicit `timeout-minutes: 10` in workflow).
- `DataCache.get()`: no timeout (SQLite is local; < 100 ms expected).
- yfinance fallback: existing timeout behavior unchanged.

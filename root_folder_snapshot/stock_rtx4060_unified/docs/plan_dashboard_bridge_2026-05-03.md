## Phase 1: Business Review

### 1.1 Problem Definition

Current state: `stock_pred_v5.jsx` is a standalone React dashboard that fetches market data in the browser and computes simulated model/backtest values locally.

Target state: the dashboard can read audited recommendation outputs generated by `stock_rtx4060_unified` without introducing broker execution, account access, or a mandatory local web server.

Quantified impact:

| Area | Current | Target |
|---|---|---|
| Dashboard data source | Browser Yahoo proxy plus synthetic fallback | Existing mode plus backend report snapshot mode |
| Python output | Markdown, JSON, audit JSONL under `--output-dir` | Same outputs plus dashboard-friendly snapshot |
| Runtime boundary | CLI report-only | CLI report-only preserved |
| Files expected to change after approval | 0 in planning phase | Estimated 4-7 files in implementation phase |
| New external service | None | None for Option A |
| Broker/order execution | Not present | Must remain not present |

### 1.2 Proposed Options

| Option | Description | Effort Days | Risk | Cost AED |
|---|---|---:|---|---:|
| A | File-based report bridge. Convert existing recommendation JSON into a dashboard snapshot and let the dashboard load/import that file. | 1-2 | Low | 0 direct software cost |
| B | Local read-only API. Add a local API server for dashboard reads such as recommendations and audit logs. | 3-5 | Medium | 0 direct software cost |
| C | Full dashboard app package. Move the JSX into a real React/Vite project and connect it to generated reports. | 5-8 | Medium-High | 0 direct software cost |

### 1.3 Recommendation and Rationale

Recommended option: Option A, file-based report bridge.

Reason 1: it preserves the current CLI report-only architecture documented in `docs/SPEC.md`.

Reason 2: it reuses existing outputs from `RecommendationEngine.write_reports()` and `audit_log.jsonl`.

Reason 3: it avoids adding a server, port, credential path, or broker-facing surface.

Rollback strategy: remove the generated dashboard snapshot command/file path and keep the existing recommendation Markdown, JSON, and audit JSONL flow unchanged.

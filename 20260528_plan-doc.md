## A. Executive Summary

## B. Context & Requirements

### B1. 문제 정의

stock_1901은 P0–P8 구현이 완성되었으나 자동화 루프의 마지막 3개 연결 고리가 열려 있다:

1. **오케스트레이션 공백**: flows/는 실행 가능하나 cron 스케줄, 의존성 관리, 알림이 없어 매일 수동 실행 필요
2. **LLM 어드바이저 미활성화**: `use_langgraph=False` 기본값으로 단일 sequential 경로만 사용. Bull/Bear/Devil 병렬 관점 부재
3. **실시간 시세 부재**: Flask 동기 API는 WebSocket 지원 불가. websockets 라이브러리 설치되어 있으나 미사용

### B2. 페르소나

| 페르소나 | 역할 | 니즈 |
|----------|------|------|
| **운영자 (본인)** | 시스템 관리 | 매일 flows 자동 실행, 실패 Slack 알림 수신 |
| **리서처** | 백테스트/분석 | research_weekly 자동 실행, MLflow 결과 자동 기록 |
| **트레이더 (미래)** | 주문 결정 | 실시간 가격 대시보드, LLM 멀티관점 분석 |

### B3. 유저 스토리

| ID | 역할 | 요구 | 수용 기준 |
|----|------|------|-----------|
| US-1 | 운영자 | KRX 장 시작 30분 전 자동 실행 | `daily_krx_flow` cron `30 8 * * 1-5` KST 실행 확인 |
| US-2 | 운영자 | flow 실패 시 Slack DM | 실패 후 30초 내 Slack 메시지 수신 |
| US-3 | 리서처 | LLM 3관점(강세/약세/악마)으로 종목 분석 | `advisor.jsonl`에 `bull_score`, `bear_score`, `devils_score` 3 필드 기록 |
| US-4 | 트레이더 | 대시보드에서 실시간 주가 확인 | WS 연결 후 1초 이내 첫 tick 수신 |
| US-5 | 운영자 | CI에서 Prefect 자동 배포 | PR merge → CI → `prefect deploy --all` 완료 <60초 |

### B4. 요구사항

#### Functional
- FR-1: Prefect `prefect.yaml` 정의, 3개 flow 배포
- FR-2: Slack/Discord 알림 (flow 완료/실패)
- FR-3: LangGraph 3-에이전트 병렬 StateGraph
- FR-4: `use_langgraph` 기본값 `True` (env flag `STOCK_LLM_LANGGRAPH=0` 으로 끄기)
- FR-5: FastAPI 1:1 엔드포인트 마이그레이션
- FR-6: `/ws/quote/{ticker}` WebSocket 엔드포인트
- FR-7: React 대시보드 WS 클라이언트

#### Non-Functional
- NFR-1: 커버리지 ≥85% CI gate 유지 (Invariant #8)
- NFR-2: `screening_output_only=True` 모든 출력 유지 (Invariant #4)
- NFR-3: audit log additive only (Invariant #3)
- NFR-4: `as_of` PIT guard 유지 (Invariant #6)

#### Non-Goals
- Kubernetes/Docker work pool (로컬 process pool MVP 충분)
- prefect.io Cloud 유료 플랜 (self-hosted)
- Alpaca 외 WS provider (MVP: Alpaca/yfinance)
- Redis pub/sub (asyncio 충분)
- 자동 주문 실행 (screening_output_only 유지)

### D2. 데이터 흐름

```
[Prefect 스케줄 흐름]
Prefect Scheduler → Worker (stock-local-pool)
  → daily_krx_flow() / daily_us_flow() / research_weekly_flow()
  → recommendation_engine → DuckDB PIT Lake → MLflow
  → notify_completion() → Slack

[실시간 시세 흐름]
클라이언트 WS connect → FastAPI /ws/quote/{ticker}
  → asyncio loop: AlpacaAdapter.get_quote(ticker) | yfinance.Ticker.fast_info
  → send_json {ticker, price, ts} every 1s

[LangGraph 흐름]
/api/recommend → orchestrator.use_langgraph=True
  → build_advisor_graph().invoke({ticker, factors})
  → [bull_agent, bear_agent, devils_agent] 병렬 실행
  → synthesize_scores → advisory_score ∈ [-1,+1]
  → audit_log/advisor.jsonl {langgraph_trace_id, ...}
```

- `as_of` guard: lake miss 시 RuntimeError (불변)
- `screening_output_only=True`: RecommendationResult 모든 출력 (불변)
- LLM 점수: GREEN→AMBER downgrade만 가능, RED/AMBER→GREEN 불가 (불변)

## J. Error Handling & Recovery

### J1. 에러 분류

| 범주 | 예시 | 처리 |
|------|------|------|
| 4xx (클라이언트) | 잘못된 ticker, 파라미터 누락 | FastAPI HTTPException(400/422) |
| 5xx (서버) | DuckDB PIT miss, MLflow 연결 실패 | 500 + loguru ERROR + Slack alert |
| 외부 API | Alpaca WS 연결 실패, Slack 웹훅 실패 | Retry 3회 + yfinance fallback |
| LLM | claude-opus-4-7 rate limit, timeout | Retry 2회 (exponential 2s/4s) + 기존 sequential path fallback |
| Prefect | Worker 미실행, Secret Block 없음 | 시작 전 health check + 명시적 에러 메시지 |

### J2. Retry/Timeout 정책

| 작업 | Retry | Timeout | Backoff |
|------|-------|---------|---------|
| Slack 알림 | 3회 | 5초 | linear 1초 |
| Alpaca WS 재연결 | 5회 | 10초 | exponential 2× |
| LLM API 호출 | 2회 | 30초 | exponential 2s/4s |
| DuckDB 쿼리 | 1회 (no retry) | 60초 | — |
| FastAPI /api/* | 0회 (fail-fast) | 30초 | — |

### J3. Idempotency

- Prefect flow 재실행: `lookback_days` 파라미터로 동일 날짜 재처리 안전
- LangGraph 재실행: `langgraph_trace_id`로 중복 advisor.jsonl 항목 감지 가능
- WS 재연결: 클라이언트 `onclose` → 3초 후 자동 재연결 (React useEffect cleanup)

### J4. UX 에러 메시징

| 상황 | UI 표시 |
|------|---------|
| WS 연결 실패 | "연결 실패 — 재시도 중..." (빨간 인디케이터) |
| 추천 API 오류 | "추천 로드 실패 — 새로고침 하세요" (토스트) |
| LangGraph timeout | "어드바이저 분석 지연 — 기본 모드로 전환됨" |

### BEST 1 — Flask Order API 엔드포인트

**버킷**: Architecture/Modularity  **PriorityScore**: 6.25 → 의존성 최상위이므로 1위 선정

#### Goal
- `api_server.py`에 주문 제출/조회/취소 REST 엔드포인트 추가
- 기존 `OrderRouter` + `compliance.py`를 API 계층에 연결
- `screening_output_only=True` 플래그 보존; 주문은 별도 `/api/order/*` 네임스페이스

#### Non-goals
- 자동 주문 실행 (수동 확인 없는 auto-submit 불가)
- Margin/Options 엔드포인트
- Webhook 기반 자동 트리거

#### Proposed Design

```
POST /api/order/preview          ← compliance 사전 검증만, 주문 미발송
POST /api/order/submit           ← 실제 주문 (requires X-Confirm: true header)
GET  /api/order/{order_id}       ← 주문 상태 조회
DELETE /api/order/{order_id}     ← 주문 취소
GET  /api/order/history          ← 최근 50건
GET  /api/account/summary        ← 계좌 잔고 / 매수여력 / 포지션
```

**`/api/order/submit` 흐름:**
```
Request → kill_switch check → compliance.check_order() →
→ COMPLIANCE OK? → OrderRouter.submit_order() → 감사 로그 기록 → Response
         ↓ NO
       ComplianceError 400
```

**감사 로그**: `audit_log/orders.jsonl` — `{ts, ticker, side, qty, price_limit, broker, order_id, status, compliance_ok, user_confirmed}`

#### PR Plan (3 PRs)

| PR | 범위 | 변경 파일 | 롤백 |
|----|------|----------|------|
| PR-1 | `/api/order/preview` + `/api/account/summary` (read-only) | `api_server.py` | 엔드포인트 삭제 |
| PR-2 | `/api/order/submit` + `/api/order/{id}` + kill-switch check | `api_server.py`, `audit_log/orders.jsonl` | 엔드포인트 삭제, KILLED 파일 생성 |
| PR-3 | `/api/order/history` + `/api/order/{id}/cancel` | `api_server.py` | 엔드포인트 삭제 |

#### Tests

- Unit: `test_order_api_kill_switch.py` — KILLED 파일 존재 시 503 반환

#### Goal

- 다수 종목 일괄 발주

#### Proposed Design

```jsx
// RecommendationCard.jsx 확장
<OrderButton
  ticker={rec.ticker}
  entry={rec.entry}
  stop={rec.stop}
  qty={rec.suggested_qty}
  onConfirm={handleOrderConfirm}   // POST /api/order/submit with X-Confirm: true
  onPreview={handleOrderPreview}   // POST /api/order/preview (no order placed)
/>

// 확인 모달 (OrderConfirmModal.jsx)
// 표시 항목: Ticker, Side, Qty, Entry, Stop, TP2, R/R, Risk budget, Broker mode (PAPER/LIVE)
// 빨간 텍스트: "This is a REAL order. Verify all fields." (LIVE 모드에만 표시)
// 버튼: [Cancel] [Confirm — Place Order]

// 대시보드 상단 KillSwitch 버튼
<KillSwitchButton onActivate={() => fetch('/api/kill-switch', {method:'POST'})} />
```

# UIUX

The current user experience is CLI plus Markdown/JSON/CSV reports. A browser dashboard is not implemented.

## Current User Flow

```mermaid
flowchart LR
    Operator[Operator] --> Command[run.ps1 or python main.py]
    Command --> CLI[CLI output]
    Command --> Recommend[recommend or ops-v1]
    Recommend --> Report[Markdown/JSON report files]
    Recommend --> Approval[approval_journal_template.csv]
    Recommend --> Zero[zero_log.md/csv]
    Report --> Review[Manual review]
    Approval --> Journal[Manual journal entry]
```

## Ops v1 Workflow

`ops-v1` implements the executable workflow from `uiux2.md`:

```text
Universe setup
-> live or synthetic data collection
-> candidate scoring
-> validation gates
-> Track-S / Track-L verdict
-> risk plan
-> daily brief
-> manual approval template
-> ZERO log
```

The workflow stops at manual review. It does not place broker orders.

## Output Surfaces

| Surface | Current Status |
|---|---|
| CLI help | Available through `python main.py --help`. |
| Self-test output | Available through `.\run.ps1 self-test`. |
| Recommendation reports | Markdown and JSON files under `reports/`. |
| Ops v1 daily brief | Markdown file generated by `.\run.ps1 ops-v1 ...`. |
| Approval journal template | CSV file generated by `.\run.ps1 ops-v1 ...`; all rows start as `REVIEW_PENDING`. |
| ZERO log | Markdown and CSV files that record blocked actions such as auto-buy and broker orders. |
| Browser dashboard | Not implemented in the unified folder. |
| Broker order screen | Not implemented and out of scope. |

The report files are the main review surface. They should stay readable without requiring a web app.

## Safe Operating Boundary

| Action | Status |
|---|---|
| Candidate screening | Allowed |
| Track-S / Track-L split | Allowed |
| Risk plan generation | Allowed |
| Manual approval template | Allowed |
| Journal entry after manual review | Allowed |
| Broker order execution | ZERO |
| Auto-buy | ZERO |
| Margin/options | ZERO |

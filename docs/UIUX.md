# UIUX

The current user experience is CLI plus Markdown/JSON/CSV reports plus a report-only dashboard bridge. The dashboard can load `dashboard_snapshot.v1` files through FILE mode or request local review data through the Flask API. It does not provide broker execution or live trading controls.

## Current User Flow

```mermaid
flowchart LR
    Operator[Operator] --> Command[run.ps1 or python main.py]
    Command --> CLI[CLI output]
    Command --> Recommend[recommend or ops-v1]
    Recommend --> Report[Markdown/JSON report files]
    Recommend --> Snapshot[dashboard_snapshot.json]
    Recommend --> Approval[approval_journal_template.csv]
    Recommend --> Zero[zero_log.md/csv]
    Snapshot --> Dashboard[REC tab in stock-pred-v5]
    Recommend --> LocalAPI[/api/recommend]
    LocalAPI --> Dashboard
    Report --> Review[Manual review]
    Dashboard --> Review
    Approval --> Journal[Manual journal entry]
```

## Ops v1 Workflow

`ops-v1` implements the executable workflow from `uiux2.md`:

```text
Universe setup
-> live or synthetic data collection
-> candidate scoring
-> validation gates
-> Track-S / Track-L verdict
-> risk plan
-> daily brief
-> manual approval template
-> ZERO log
```

The workflow stops at manual review. It does not place broker orders.

## Output Surfaces

| Surface | Current Status |
|---|---|
| CLI help | Available through `python main.py --help`. |
| Self-test output | Available through `.\run.ps1 self-test`. |
| Recommendation reports | Markdown and JSON files under `reports/`. |
| Ops v1 daily brief | Markdown file generated by `.\run.ps1 ops-v1 ...`. |
| Approval journal template | CSV file generated by `.\run.ps1 ops-v1 ...`; all rows start as `REVIEW_PENDING`. |
| ZERO log | Markdown and CSV files that record blocked actions such as auto-buy and broker orders. |
| Browser dashboard bridge | Available through `dashboard-export`, the legacy `dashboard/stock_pred_v5.jsx`, and the full Vite workspace `root_folder_snapshot/stock-pred-v5`; REC can use FILE mode or local API mode. |
| Broker order screen | Not implemented and out of scope. |

The report files are the main review surface. They should stay readable without requiring a web app.

## Safe Operating Boundary

| Action | Status |
|---|---|
| Candidate screening | Allowed |
| Track-S / Track-L split | Allowed |
| Risk plan generation | Allowed |
| Manual approval template | Allowed |
| Journal entry after manual review | Allowed |
| Broker order execution | ZERO |
| Auto-buy | ZERO |
| Margin/options | ZERO |

## Command

```powershell
.\run.ps1 ops-v1 --period 3y --top 5 --full --prefer-gpu --model-kind xgb --cv-gap 5 --output-dir reports/ops_v1
```

Dashboard snapshot workflow:

```powershell
.\run.ps1 recommend --synthetic --universe "SYNTH-A,SYNTH-B" --top 2 --model-kind logistic --cv-gap 5 --output-dir reports/dashboard_bridge_smoke
.\run.ps1 dashboard-export --recommendation-json reports/dashboard_bridge_smoke/recommendations_algo_v2_YYYYMMDD_HHMMSS.json --output reports/dashboard_bridge_smoke/dashboard_snapshot.json
node dashboard\verify_bridge_smoke.mjs
```

Observed browser evidence is recorded in `reports/dashboard_browser_verification/dashboard_browser_verification.md`.

Current local dashboard workflow:

```powershell
python api_server.py --port 5151
cd root_folder_snapshot\stock-pred-v5
npm run dev
```

In API mode, the REC panel requests `/api/recommend`; SIGNAL and MODELS request `/api/model-scores`.
